# Зачётное задание. Микрообласть с нуля (10 минут)

**Система:** CTMSN (https://ctmsn.ru)

## Цель

За одну сессию построить минимальную семантическую сеть **в пустом рабочем пространстве**, задать параметризацию, проверить формулы в трёхзначной логике Клини и выполнить операции форсирования.

## Условия

| Параметр | Значение |
|----------|----------|
| Подготовка | не требуется; допускается только знание интерфейса |
| Исходное состояние | новое рабочее пространство **без** загрузки готовых сценариев |
| Время | не более **10 минут** с момента входа в пространство |
| Идентификаторы концептов | латиница, `snake_case` |

## Минимальные объёмы

Выполните **все** пункты; числа — нижняя граница.

| Элемент | Минимум |
|---------|---------|
| Концепты | 4 |
| Предикаты | 2 (арность 2) |
| Факты | 4 |
| Переменные с перечислимым доменом | 1 |
| Формулы (с вычислением) | 3 |
| Операции форсирования | 2 различных типа из списка: `check`, `forces`, `force` |

**Связность:** каждый концепт участвует хотя бы в одном факте.

## Порядок выполнения (контрольный лист)

1. **Вход.** Авторизуйтесь, создайте рабочее пространство, откройте редактор. **Не загружайте** сценарии из списка.
2. **Сеть.** Создайте концепты, предикаты и факты в объёмах из таблицы выше.
3. **Параметризация.** Заведите одну переменную; домен — подмножество уже созданных концептов (не менее двух значений). Контекст на старте: переменная **не назначена** (если интерфейс позволяет снять назначение).
4. **Формулы.** Создайте и **вычислите**:
   - `FactAtom` по существующему факту — ожидается **TRUE**;
   - `FactAtom` по отсутствующей связи между концептами из вашей сети — ожидается **FALSE**;
   - формула, в которой в атоме участвует **неназначенная** переменная, — ожидается **UNKNOWN**.
5. **Форсирование.** Выполните две операции из тройки `check` / `forces` / `force` (разные типы), согласованные с вашими формулами и контекстом. Названия условий и целевой формулы φ должны быть согласованы с построенной сетью.
6. **Устно (30–60 с).** Ответьте на один вопрос преподавателя: чем отличается результат **FALSE** у атомарной формулы от **UNKNOWN** в вашей модели.

## Эталонная микрообласть (по желанию)

Если нет собственной идеи, используйте схему **«Коворкинг»** (можно переименовать концепты, сохранив структуру).

| Концепты | `alice`, `bob`, `desk_1`, `room_x` |
|----------|-------------------------------------|
| Предикаты | `uses(person, desk)`, `located_in(desk, room)` |
| Факты (пример) | `uses(alice, desk_1)`, `uses(bob, desk_1)`, `located_in(desk_1, room_x)` — добавьте четвёртый факт, чтобы выполнить минимум и связность |

## Критерии зачёта

| Критерий | Требование |
|----------|------------|
| Чистый лист | Не использовались загрузки готовых сценариев до завершения шагов 2–5 |
| Объём | Выполнены минимальные числа из таблицы «Минимальные объёмы» |
| Логика | Три вычисления дают **TRUE**, **FALSE**, **UNKNOWN** согласованно с контекстом |
| Форсинг | Две операции разных типов выполнены и объяснимы по вашей сети |
| Время | Работа укладывается в 10 минут |
