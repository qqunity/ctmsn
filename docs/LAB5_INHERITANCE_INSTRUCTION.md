## 2 Теоретические сведения

### 2.1 Информация, знание и представление знаний

Информация — структурированные данные, осмысленные в определённом контексте. Знание — обоснованное истинное мнение, выраженное в форме утверждений о мире. Переход от данных к знаниям предполагает уровни абстракции: данные → информация → знание → мудрость.

Центральная проблема ИИ — **приобретение и представление знаний**. Тексты, таблицы и базы данных содержат знания в неструктурированной форме. Семантические сети предоставляют формальный аппарат для перехода от текстового описания к вычислимой структуре. При этом возникает «узкое место приобретения знаний»: извлечение отношений из текста требует экспертизы и формализации.

В контексте CTMSN знание представляется тремя уровнями:

| Уровень          | Элемент CTMSN      | Пример                         |
|------------------|--------------------|---------------------------------|
| Данные           | Концепт            | `penguin` — идентификатор       |
| Информация       | Факт               | `isa(penguin, bird)`            |
| Знание           | Формула + форсинг  | `∀x: bird(x) → can_fly(x)`     |

### 2.2 Иерархии и наследование

**ISA-отношение** (is-a) — фундаментальный вид связи в семантических сетях, задающий отношение «подкласс — надкласс». Граф ISA-отношений образует иерархию (направленный ациклический граф):

```
         animal
        /      \
     bird      fish
    /    \       \
penguin  sparrow  salmon
```

**Наследование свойств** — механизм, при котором подкласс автоматически получает свойства надкласса. Если `has_ability(bird, fly)` и `isa(sparrow, bird)`, то по умолчанию `has_ability(sparrow, fly)`.

**Исключения** нарушают наследование по умолчанию: пингвин является птицей, но не летает. Это пример **немонотонного рассуждения** — добавление новой информации (пингвин — птица, но не летает) отменяет ранее сделанный вывод (все птицы летают).

В CTMSN наследование не является автоматическим: каждый факт должен быть задан явно. Это позволяет точно контролировать, какие свойства распространяются, а какие — нет. Конфликт между унаследованным свойством и исключением моделируется через пару предикатов: `has_ability` (свойство по умолчанию) и `lacks_ability` (явное исключение).

### 2.3 Кванторы в семантических сетях

В классической логике предикатов кванторы связывают переменные:

| Квантор        | Обозначение | Чтение                                |
|----------------|-------------|---------------------------------------|
| Универсальный  | ∀x: P(x)   | «Для всех x выполняется P(x)»        |
| Экзистенциальный | ∃x: P(x) | «Существует x, для которого P(x)»    |

В CTMSN кванторы формально не реализованы, однако их семантика может быть приближённо воспроизведена через параметризацию и форсинг:

**Экзистенциальный квантор ∃.** Операция `force()` перебирает все допустимые значения переменных из домена и ищет хотя бы одно присваивание, при котором формула истинна. Если `force()` возвращает TRUE и найденный контекст, это эквивалентно утверждению «существует значение переменной, при котором формула истинна».

**Универсальный квантор ∀.** Прямой аналог отсутствует. Для проверки универсального утверждения необходимо перебрать все значения домена вручную и убедиться, что формула истинна при каждом из них. Если хотя бы для одного значения формула ложна, универсальное утверждение опровергнуто.

**Ограничение.** Данный подход работает только для конечных перечислимых доменов (`EnumDomain`, `RangeDomain`). Для `PredicateDomain` перебор невозможен, и результат остаётся UNKNOWN. Кроме того, при большом числе переменных и значений декартово произведение может превысить порог `max_branch`, что также приводит к UNKNOWN.

### 2.4 Дескрипции и идентичность

**Определённая дескрипция** — выражение вида «единственный x, обладающий свойством P». Например, «автор, который преподаёт базы данных» — это определённая дескрипция, выделяющая конкретного индивида из множества.

В CTMSN определённая дескрипция моделируется как комбинация переменной с доменом и формулы-ограничения. Если `force()` находит ровно одно значение переменной, удовлетворяющее формуле, — дескрипция успешна. Если подходящих значений нет или их несколько — дескрипция неопределена.

**Индивиды и виртуальные индивиды.** Индивид — конкретная сущность, обозначаемая уникальным идентификатором (концепт). Виртуальный индивид — сущность, описываемая через свойства, без явного имени. Переменная в CTMSN может выступать как виртуальный индивид: значение переменной определяется контекстом, а не задано заранее.

**Идентичность.** Формула `EqAtom(a, b)` проверяет, ссылаются ли два терма на один и тот же концепт. Важные случаи:

| Терм A      | Терм B      | Результат | Пояснение                              |
|-------------|-------------|-----------|----------------------------------------|
| tux         | tux         | TRUE      | Один и тот же концепт                  |
| tux         | jack        | FALSE     | Разные концепты                        |
| individual  | tux         | UNKNOWN   | Переменная `individual` не назначена   |

**Подстановочность равных (Лейбниц).** Если `a = b`, то в любом контексте `a` можно заменить на `b` без изменения истинности. Этот принцип нарушается в **интенсиональных контекстах** — например, в высказываниях о знании или убеждении (см. раздел 2.5).

### 2.5 Пропозициональные установки и модальные контексты

**Пропозициональная установка** — отношение субъекта к суждению: «знает, что P», «верит, что P», «хочет, чтобы P». Пропозициональные установки порождают **интенсиональные контексты**, в которых обычные правила подстановки не работают.

**Пример референциальной непрозрачности.** Пусть Утренняя звезда = Вечерняя звезда (оба названия обозначают Венеру). Высказывание «Древний астроном верит, что Утренняя звезда восходит утром» истинно. Но замена «Утренней звезды» на «Вечернюю звезду» может дать ложное высказывание: астроном мог не знать, что это одно и то же небесное тело.

В CTMSN пропозициональные установки упрощённо моделируются через предикаты типа `knows_about(agent, entity)`. Полноценное моделирование потребовало бы вложенных контекстов и операторов над пропозициями, что выходит за рамки текущей реализации.

**Лямбда-абстракция (λ-абстракция)** — механизм создания функций из выражений: `λx.P(x)` обозначает «свойство быть таким x, что P(x)». В семантических сетях λ-абстракция позволяет описывать свойства без привязки к конкретному индивиду. Параметризация в CTMSN (переменная с доменом) является ограниченным аналогом λ-абстракции: переменная `species` с доменом и формула `has_ability(species, ability_fly)` представляют свойство «способность летать для данного вида».

### 2.6 Время в семантических сетях

Время может быть представлено в семантической сети несколькими способами:

| Способ                    | Описание                                         | Пример в CTMSN                   |
|---------------------------|--------------------------------------------------|----------------------------------|
| Временные индексы         | Отдельные концепты для моментов времени          | `t1`, `t2` как узлы графа       |
| Состояния                 | Концепты, обозначающие состояния процесса        | `state_before`, `state_after`    |
| Морфизмы перехода         | Рёбра, описывающие переходы между состояниями    | `transition(state_before, state_after)` |

Сценарий `time_process` (Лабораторная работа 2) демонстрирует моделирование процесса во времени через композицию морфизмов. Режим `sun` описывает восход и закат как переходы между состояниями «ниже горизонта» и «выше горизонта».

### 2.7 Вывод при неполной информации

**Неполная информация** — ситуация, при которой имеющихся данных недостаточно для определения истинности или ложности утверждения. В CTMSN неполная информация выражается значением **UNKNOWN** в трёхзначной логике Клини.

Источники неполной информации:

| Источник                     | Проявление в CTMSN                                         |
|------------------------------|-------------------------------------------------------------|
| Неназначенная переменная     | `FactAtom("p", var, c)` → UNKNOWN при var без значения      |
| Отсутствующий факт           | `FactAtom("p", a, b)` → FALSE (предположение о замкнутости) |
| Конфликт в иерархии          | `has_ability(bird, fly)` vs `lacks_ability(penguin, fly)`   |

**Предположение о замкнутости мира (Closed World Assumption, CWA)** — если факт не представлен в сети, он считается ложным. CTMSN следует CWA: отсутствие факта `teaches(petrov, course_db)` означает FALSE, а не UNKNOWN.

**Предположение об открытости мира (Open World Assumption, OWA)** — отсутствие факта означает неизвестность, а не ложность. Трёхзначная логика Клини в CTMSN частично реализует OWA через значение UNKNOWN для неопределённых переменных.

**Наследование с исключениями.** При наличии конфликтующих свойств в иерархии (птицы летают, но пингвины — нет) требуется правило разрешения конфликтов. Типичное правило: **более специфичная информация имеет приоритет** над менее специфичной. Пингвин ближе к «не летает», чем к «летает» (наследуемому от «птица»).

**Принцип максимальной энтропии** — при отсутствии дополнительных данных следует принимать наименее ограничительную гипотезу. Применительно к семантическим сетям: если нет оснований предпочесть одну категоризацию другой, следует считать все варианты равновероятными.

### 2.8 Семантическая безопасность

**Извлечение знаний из данных** — процесс формирования структурированных утверждений на основе неструктурированных источников (текстов, таблиц, журналов). Качество результата зависит от полноты источника и корректности формализации.

**Задача о продвижении информации** — проблема распространения утверждений в сети агентов, где каждый агент интерпретирует и ретранслирует информацию. Искажения накапливаются, и исходное утверждение может существенно измениться.

**Пост-правда и фейковые новости.** В контексте семантических сетей фейковая информация — это факт, внесённый в сеть без обоснования. Если `FactAtom("teaches", petrov, course_db)` добавлен в сеть без проверки, все зависящие от него формулы могут дать ошибочный результат. Форсирование позволяет обнаружить противоречия: если добавленный факт нарушает условия (check возвращает ok=False), это сигнализирует о возможной недостоверности.

**Семантическая безопасность** — свойство системы представления знаний, гарантирующее, что выводимые утверждения соответствуют исходным данным и правилам вывода. Механизм форсирования в CTMSN обеспечивает элемент семантической безопасности: операция `check` верифицирует выполнение ограничений, а `forces` гарантирует, что вывод следует из контекста.

### 4.4 Наследование и его ограничения

1. Создайте формулу `sparrow_flies`: `FactAtom("has_ability", sparrow, ability_fly)`.
2. Вычислите — ожидается результат **FALSE**.
3. Объяснение: факт `has_ability(bird, ability_fly)` описывает способность класса «птица», но НЕ распространяется автоматически на подклассы и экземпляры. В CTMSN каждый факт задаётся явно.
4. Создайте формулу `bird_flies`: `FactAtom("has_ability", bird, ability_fly)`.
5. Вычислите — ожидается результат **TRUE**.
6. Зафиксируйте наблюдение: свойство задано для класса `bird`, но отсутствует для его подкласса `sparrow`. Это демонстрирует отсутствие автоматического наследования.

### 4.5 Моделирование конфликта в иерархии

1. Создайте формулу `bird_flies_and_penguin_not`:

```
And(
  FactAtom("has_ability", bird, ability_fly),
  FactAtom("lacks_ability", penguin, ability_fly)
)
```

2. Вычислите — ожидается результат **TRUE** (T ∧ T = T). Оба факта присутствуют в сети одновременно.
3. Создайте формулу-импликацию `if_bird_then_flies`:

```
Implies(
  FactAtom("isa", penguin, bird),
  FactAtom("has_ability", penguin, ability_fly)
)
```

4. Вычислите — ожидается результат **FALSE** (T → F = F). Пингвин является птицей, но не обладает способностью летать. Импликация ложна — это формальная фиксация исключения из правила наследования.

### 4.6 Моделирование экзистенциального квантора (∃)

Задача: «Существует ли вид из домена `species`, который обладает способностью плавать?»

1. Создайте формулу `species_can_swim`: `FactAtom("has_ability", species, ability_swim)`.
2. Вычислите — ожидается результат **UNKNOWN** (переменная `species` не назначена).
3. Выполните операцию **force**:
   - В панели «Форсирование» выберите целевую формулу `species_can_swim`.
   - Условия оставьте пустыми.
   - Нажмите **«Форсировать»**.
4. Ожидаемый результат: **TRUE**. Система нашла значение `species = penguin` (факт `has_ability(penguin, ability_swim)` присутствует в сети).
5. Зафиксируйте: операция `force()` реализует аналог экзистенциального квантора — она доказывает, что **существует** значение переменной, при котором формула истинна.

### 4.7 Моделирование универсального квантора (∀)

Задача: «Верно ли, что все виды из домена `species` являются подклассами `animal`?»

1. Создайте формулу `species_is_animal`: `FactAtom("isa", species, animal)`.
2. Назначьте переменной `species` значение `penguin`. Вычислите — ожидается **FALSE** (в сети нет прямого факта `isa(penguin, animal)`, есть только `isa(penguin, bird)`).
3. Назначьте `species = sparrow`. Вычислите — ожидается **FALSE** (аналогично).
4. Назначьте `species = salmon`. Вычислите — ожидается **FALSE** (есть только `isa(salmon, fish)`).
5. Вывод: универсальное утверждение «∀species: isa(species, animal)» **ложно** при прямой проверке, поскольку ISA-отношение не транзитивно в CTMSN.
6. Теперь проверьте другое утверждение — `species_is_bird`: `FactAtom("isa", species, bird)`.
7. Назначьте `species = penguin`. Вычислите — **TRUE** (факт `isa(penguin, bird)` есть).
8. Назначьте `species = sparrow`. Вычислите — **TRUE** (факт `isa(sparrow, bird)` есть).
9. Назначьте `species = salmon`. Вычислите — **FALSE** (лосось — не птица).
10. Вывод: универсальное утверждение «∀species: isa(species, bird)» **ложно** — контрпримером является `salmon`.
11. Снимите назначение переменной `species` для продолжения работы.

### 4.8 Идентичность и дескрипции

**Идентичность:**

1. Создайте формулу `tux_is_tux`: `EqAtom(tux, tux)`.
2. Вычислите — ожидается **TRUE** (рефлексивность: каждый концепт равен самому себе).
3. Создайте формулу `tux_is_jack`: `EqAtom(tux, jack)`.
4. Вычислите — ожидается **FALSE** (различные концепты).
5. Создайте формулу `individual_is_tux`: `EqAtom(individual, tux)`.
6. Вычислите — ожидается **UNKNOWN** (переменная `individual` не назначена).

**Определённая дескрипция:**

Задача: «Найти индивида, который является пингвином» — определённая дескрипция «единственный x такой, что instance_of(x, penguin)».

7. Создайте формулу `find_penguin`: `FactAtom("instance_of", individual, penguin)`.
8. Выполните операцию `force` с целевой формулой `find_penguin`.
9. Ожидаемый результат: **TRUE**, найден `individual = tux`. Система нашла единственного индивида из домена, удовлетворяющего дескрипции.
10. Зафиксируйте: `force()` реализует поиск по определённой дескрипции в конечном домене.

### 4.9 Пропозициональные установки

1. Создайте формулу `jack_knows_tux`: `FactAtom("knows_about", jack, tux)`.
2. Вычислите — ожидается **TRUE**.
3. Создайте формулу `jack_knows_nemo`: `FactAtom("knows_about", jack, nemo)`.
4. Вычислите — ожидается **FALSE**.
5. Обратите внимание на ограничение модели: предикат `knows_about(jack, tux)` фиксирует лишь факт осведомлённости, но не позволяет выразить, **что именно** Джек знает о Туксе. Для этого потребовались бы вложенные пропозиции и операторы типа `knows_that(agent, proposition)`.
6. Рассмотрите ситуацию с точки зрения подстановочности равных. Допустим, `tux = mystery_penguin` (Тукс и «загадочный пингвин» — одна и та же сущность). Из `knows_about(jack, tux)` и `tux = mystery_penguin` формально следует `knows_about(jack, mystery_penguin)`. Но в реальности Джек может не знать, что Тукс — это загадочный пингвин. Это — **референциальная непрозрачность** интенсионального контекста.

### 4.10 Вывод при неполной информации

1. Убедитесь, что обе переменные (`species`, `individual`) **не назначены**.
2. Создайте формулу `enrolled_or_swim`:

```
Or(
  FactAtom("has_ability", species, ability_swim),
  FactAtom("instance_of", individual, penguin)
)
```

3. Вычислите — ожидается **UNKNOWN** (U ∨ U = U: обе подформулы неопределены).
4. Назначьте `individual = tux`. Вычислите — ожидается **TRUE** (U ∨ T = T: правая часть дизъюнкции истинна, и результат не зависит от левой).
5. Снимите назначение `individual`. Назначьте `species = penguin`. Вычислите — ожидается **TRUE** (T ∨ U = T: левая часть истинна).
6. Зафиксируйте: в трёхзначной логике Клини частичное назначение переменных может быть достаточным для определения результата. Это — ключевое свойство для вывода при неполной информации.

### 4.11 Эксперимент: верификация информации

Смоделируйте ситуацию внесения недостоверного факта в сеть.

1. Создайте условие `cond_penguin_bird`: `FactAtom("isa", penguin, bird)` — истинное ограничение.
2. Создайте условие `cond_penguin_flies`: `FactAtom("has_ability", penguin, ability_fly)` — ложное ограничение (пингвины не летают).
3. Выполните операцию **check** с обоими условиями.
4. Ожидаемый результат: **ok=False**, нарушено условие `cond_penguin_flies`.
5. Зафиксируйте: механизм `check` обнаружил противоречие — факт `has_ability(penguin, ability_fly)` отсутствует в сети. Если бы такой факт был ошибочно добавлен (фейковая информация), операция `check` с условием `lacks_ability(penguin, ability_fly)` выявила бы конфликт.
