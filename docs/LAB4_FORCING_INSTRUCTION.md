## 2 Теоретические сведения

### 2.1 Параметризация

Параметризация позволяет вводить в семантическую сеть неопределённость. Вместо конкретных концептов в формулах используются **переменные** — именованные слоты, принимающие значения из заданного **домена**.

**Переменная (Variable)** описывается тремя атрибутами:

| Атрибут  | Описание                                           |
|----------|----------------------------------------------------|
| name     | Уникальное имя переменной                          |
| domain   | Множество допустимых значений                       |
| type_tag | Необязательная метка типа                           |

**Домен (Domain)** определяет множество значений, которые переменная может принимать:

| Тип домена      | Описание                                          | Пример                           |
|------------------|--------------------------------------------------|----------------------------------|
| EnumDomain       | Конечное перечисление значений                    | {petrov, ivanov}                 |
| RangeDomain      | Диапазон числовых значений (целые при перечислении)| [1, 10]                          |
| PredicateDomain  | Произвольное условие-предикат (не перечислим)     | λx: x.tag == "student"           |

### 2.2 Контекст

**Контекст (Context)** — частичное присваивание значений переменным. Контекст может быть:

- **Пустым** — ни одна переменная не назначена.
- **Частичным** — часть переменных имеет значения, остальные не определены.
- **Полным** — все переменные формулы назначены.

Операции над контекстом:

| Операция    | Описание                                                   |
|-------------|------------------------------------------------------------|
| set(v, val) | Назначить значение переменной (с проверкой домена)         |
| get(v)      | Получить текущее значение переменной (или None)            |
| is_assigned | Проверить, назначена ли переменная                          |
| extend      | Создать новый контекст, дополненный новыми назначениями    |

### 2.3 Условия (Conditions)

**Условия** — набор формул, которые должны выполняться одновременно. Условия задают ограничения на допустимые состояния сети. Например, условие `Not(FactAtom("teaches", petrov, course_db))` запрещает ситуацию, в которой Петров преподаёт курс «Базы данных».

### 2.4 Механизм форсирования (Forcing)

Форсирование — центральный механизм вывода в CTMSN. **ForcingEngine** предоставляет три операции:

**check(ctx, conditions)** — проверяет, выполняются ли все условия в данном контексте:

| Результат | Описание                                                    |
|-----------|-------------------------------------------------------------|
| ok=True   | Все условия имеют значение TRUE или UNKNOWN                 |
| ok=False  | Хотя бы одно условие имеет значение FALSE                  |

Дополнительно возвращаются списки `violated` (нарушенные условия) и `unknown` (неопределённые условия).

**forces(ctx, phi, conditions)** — проверяет, форсирует ли контекст формулу φ при заданных условиях:

| Результат | Описание                                                        |
|-----------|-----------------------------------------------------------------|
| TRUE      | Условия выполнены, формула φ истинна, неопределённых условий нет |
| FALSE     | Условия нарушены или формула φ ложна                            |
| UNKNOWN   | Недостаточно информации для определения результата              |

**force(ctx, phi, conditions)** — автоматический поиск расширения контекста, при котором контекст форсирует φ:

| Результат | Описание                                                                  |
|-----------|---------------------------------------------------------------------------|
| TRUE      | Найдено расширение контекста, при котором forces возвращает TRUE           |
| FALSE     | Ни одно допустимое расширение не удовлетворяет формуле и условиям         |
| UNKNOWN   | Поиск невозможен (пространство слишком велико или домен не перечислим)    |

При успешном поиске возвращается расширенный контекст с найденными значениями переменных.

### 2.5 Стратегия поиска

Метод `force()` использует стратегию перебора **BruteEnumStrategy**:

1. Из формулы φ и условий извлекаются все переменные.
2. Определяются неназначенные переменные (отсутствующие в текущем контексте).
3. Для каждой переменной перечисляются значения домена (EnumDomain, RangeDomain).
4. Строится декартово произведение всех возможных значений.
5. Для каждой комбинации проверяется, форсирует ли расширенный контекст формулу.
6. Если пространство поиска превышает `max_branch` (по умолчанию 2000), поиск прекращается с результатом UNKNOWN.

Переменные с `PredicateDomain` пропускаются, так как их домен не является перечислимым.

## 4 Порядок выполнения

### 4.1 Вход в систему

1. Откройте в браузере сайт https://ctmsn.ru.
2. Войдите в систему, используя учётную запись, созданную ранее.
3. При необходимости зарегистрируйтесь (формат логина: `Фамилия_ИО_группа`).

### 4.2 Создание рабочего пространства

1. На странице «Мои пространства» создайте новое рабочее пространство (например, «ЛР4 — Форсирование»).
2. Перейдите в рабочую область.

### 4.3 Загрузка сценария

1. В верхней панели в выпадающем списке сценариев выберите `lab3_formulas`.
2. Нажмите кнопку **«Загрузить»**.
3. Дождитесь отображения графа семантической сети (6 узлов, 7 рёбер).
4. Убедитесь, что на панели «Переменные» отображаются три переменные: `student`, `course`, `teacher`.

### 4.4 Изучение панели переменных

1. В правой панели найдите раздел **«Переменные»**.
2. Обратите внимание на отображение каждой переменной: имя, домен (перечисление допустимых значений), текущее значение.
3. Убедитесь, что переменная `course` имеет назначенное значение `course_db`, а переменные `student` и `teacher` не назначены.
4. Попробуйте назначить переменной `student` значение `petrov` — значение должно быть принято (входит в домен).
5. Снимите назначение переменной `student` для продолжения работы.

### 4.5 Изучение справочных материалов

1. Нажмите кнопку **«?»** (справка) на панели инструментов.
2. Перейдите на вкладку **«Глоссарий»** и ознакомьтесь с определениями терминов «Контекст», «Условия», «Форсирование».
3. Перейдите на вкладку **«Логика»** и освежите в памяти таблицы истинности (материал Лабораторной работы 3).
4. Закройте справочную панель.

### 4.6 Создание формул-условий

Для работы с форсированием необходимо создать формулы, которые будут использоваться как условия и как целевая формула.

1. Создайте формулу `cond_belongs`: `FactAtom("belongs_to", dept_cs, university)`.
   - Вычислите — ожидается **TRUE** (факт присутствует в сети).
2. Создайте формулу `cond_works`: `FactAtom("works_at", ivanov, dept_cs)`.
   - Вычислите — ожидается **TRUE**.
3. Создайте формулу `cond_not_teaches_petrov`: `Not(FactAtom("teaches", petrov, course_db))`.
   - Вычислите — ожидается **TRUE** (Петров не преподаёт, отрицание FALSE даёт TRUE).

### 4.7 Создание целевой формулы

1. Создайте формулу `goal_enrolled`: `FactAtom("enrolled_in", petrov, course_db)`.
   - Вычислите — ожидается **TRUE**.
2. Создайте формулу `goal_with_var`: `FactAtom("enrolled_in", student, course_db)`.
   - Вычислите — ожидается **UNKNOWN** (переменная `student` не назначена).

### 4.8 Операция check — проверка условий

1. В правой панели найдите раздел **«Форсирование»**.
2. В поле **«Контекст»** оставьте значение «Активный контекст».
3. В списке **«Условия»** отметьте галочками формулы `cond_belongs`, `cond_works`, `cond_not_teaches_petrov`.
4. Нажмите кнопку **«Проверить»**.
5. Ожидаемый результат: **«Все условия выполнены»** (ok=True). Все три условия имеют значение TRUE в текущем контексте.
6. Обратите внимание на цветовую индикацию: каждое условие отображается с меткой TRUE (зелёный).

### 4.9 Операция forces — проверка форсирования

**Случай 1: формула TRUE (форсирование выполнено)**

1. В поле **«Целевая формула (φ)»** выберите `goal_enrolled`.
2. Оставьте отмеченными те же три условия.
3. Нажмите кнопку **«Форсирует?»**.
4. Ожидаемый результат: **TRUE** (зелёный). Контекст форсирует формулу, так как факт `enrolled_in(petrov, course_db)` присутствует в сети, все условия выполнены, неопределённых условий нет.
5. Нажмите **«Объяснение»** и изучите пошаговое обоснование результата.

**Случай 2: формула UNKNOWN (недостаточно информации)**

1. В поле **«Целевая формула (φ)»** выберите `goal_with_var`.
2. Нажмите кнопку **«Форсирует?»**.
3. Ожидаемый результат: **UNKNOWN** (жёлтый). Переменная `student` не назначена — невозможно определить, присутствует ли факт `enrolled_in(student, course_db)`.

**Случай 3: условия нарушены**

1. Создайте формулу `cond_false`: `FactAtom("teaches", petrov, course_db)`.
   - Вычислите — ожидается **FALSE**.
2. Отметьте `cond_false` в условиях (вместо или вместе с предыдущими).
3. В поле φ выберите `goal_enrolled`.
4. Нажмите **«Форсирует?»**.
5. Ожидаемый результат: **FALSE** (красный). Условие `cond_false` нарушено (FALSE), поэтому форсирование невозможно.

### 4.10 Операция force — автоматический поиск расширения контекста

Результат операции `force` отображается в блоке «Результат сценария» панели форсирования после загрузки сценария. Для наблюдения за автоматическим поиском выполните следующие шаги:

1. Убедитесь, что переменная `student` **не назначена** в контексте.
2. Загрузите (или перезагрузите) сценарий `lab3_formulas`.
3. Обратите внимание на блок **«Результат сценария»** в панели «Форсирование»:
   - **check:** ok — все условия сценария выполнены.
   - **forces:** UNKNOWN — формула не может быть определена при текущем неполном контексте.
   - **force:** TRUE — система нашла расширение контекста, при котором формула форсируется.
4. Это означает, что метод `force()` перебрал возможные значения неназначенных переменных и нашёл такое присваивание, при котором целевая формула становится истинной при выполненных условиях.

### 4.11 Эксперимент с назначением переменных

1. Назначьте переменной `student` значение `petrov`.
2. Перезагрузите сценарий.
3. Наблюдайте изменение результатов:
   - **forces** должен измениться с UNKNOWN на TRUE (все переменные назначены, формула истинна).
   - **force** должен остаться TRUE (ранний возврат — формула уже форсирована).
4. Снимите назначение `student` и назначьте `teacher` значение `ivanov`.
5. Перезагрузите и проанализируйте результаты.

### 4.12 Изучение истории операций
